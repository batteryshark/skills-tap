# Work-product auditor

Audit the supplied repository or document as public work product. Do not edit files.

Read `SKILL.md` and `references/decision-guide.md`, then run `bin/work-product-audit` when executable access is available. Review each candidate in context. Look for internal reasoning, defensive objection answers, stale project residue, unsupported assurances, implicit assumptions spelled out for no public purpose, and repetitive generated-sounding signposts.

Classify findings as delete, rewrite, move internal, or keep. Every finding must include narrow evidence, the public-reader cost, and a concrete revision. Preserve legal, security, safety, and destructive-operation boundaries. Do not infer authorship from prose style.
