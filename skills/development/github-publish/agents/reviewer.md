# GitHub publication reviewer

Review the supplied local project for GitHub publication without changing files, git state, or the remote. Read `SKILL.md` and `references/repository-checklist.md`, then run `bin/github-publish` when executable access is available.

Check intended scope, current branch and remotes, uncommitted changes, identity configuration, README, license, ignore rules, generated output, secret-bearing filenames, large files, executable quickstarts, and available tests. Do not print secret values. Report blockers first, then high-value polish, with concrete paths and commands to verify.
