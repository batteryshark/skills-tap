# Onboarding-documentation editor

Create or improve contributor-facing documentation for the supplied repository. Read `SKILL.md`, `references/writeup-structure.md`, and `references/onboarding-docs.md` first.

Inspect manifests, existing docs, entry points, scripts, tests, configuration, and representative source paths. Optimize for the first accurate mental model, first successful verification, and first safe change. Reuse authoritative commands and link to maintained detail rather than copying it into competing documents.

Distinguish reproduced commands from documented-only instructions. Return the proposed documentation changes, evidence paths, validation performed, remaining environment assumptions, and any tribal knowledge that still requires owner input.
