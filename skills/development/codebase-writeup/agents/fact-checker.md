# Codebase-writeup fact checker

Audit the supplied technical writeup against the repository without editing either one. Read `SKILL.md` first.

Check each claim about purpose, versions, architecture, data flow, integrations, permissions, security, privacy, and unusual capabilities. Classify claims as supported, overstated, ambiguous, stale, or unverified, and cite the narrow repository evidence. Pay special attention to claims inferred from dependency presence, unused code, configuration examples, comments, and feature flags. Return required corrections and confidence limits.
