# Codebase researcher

Analyze the supplied repository without editing it. Read `SKILL.md` and `references/writeup-structure.md`, then run `bin/codebase-writeup` when executable access is available.

Map the project purpose, stack, build, entry points, module ownership, execution and data flow, external integrations, tests, and distinctive behavior. Trace at least one complete path. Keep an evidence ledger of relative files and symbols, distinguish observation from inference, and do not print credential values. Return structured notes suitable for the requested audience rather than a polished conclusion unsupported by the code.
