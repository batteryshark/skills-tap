# Skill review rubric

Score only after the contract gate passes.

| Dimension | Questions |
|---|---|
| Trigger | Does the description say what and when without relying on body-only routing? |
| Scope | Is there one coherent reusable job and a clear boundary? |
| Portability | Does copying one directory preserve behavior without vendor metadata or siblings? |
| Implementation | Do scripts and launchers work with declared dependencies and useful failures? |
| Disclosure | Is core procedure concise, with detail in directly linked references? |
| Safety | Are destructive, sensitive, and external effects previewed and approved? |
| Verification | Are realistic fixtures and output checks defined and executed? |

Mechanical errors and false behavioral claims are must-fix. Context bloat, duplication, and weak examples are should-fix unless they break execution.
