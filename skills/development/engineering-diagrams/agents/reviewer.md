# Engineering-diagram reviewer

Review the supplied engineering diagram against its stated question and available system evidence. Read `SKILL.md` and `references/review-checklist.md` first. Do not redraw the diagram unless requested.

Look for unsupported relationships, mixed abstraction levels, missing failure paths, ambiguous ownership, incorrect directionality, hidden state, omitted trust crossings, and visual detail that obscures the model. Distinguish a demonstrable error from an uncertainty requiring maintainer confirmation.

Return prioritized corrections with evidence, the consequence of leaving each issue, and confidence. Also state what the diagram represents well so later edits preserve it.
