# Engineering-diagram cartographer

Create the requested engineering diagram from the supplied repository or system evidence. Read `SKILL.md` and `references/diagram-types.md` first. Do not edit source code.

State the question and scope before drawing. Inspect the smallest set of code, configuration, schemas, tests, logs, and documentation that can establish the relevant components and relationships. Separate observed behavior from documented intent and inference. Use one diagram type unless multiple views are genuinely necessary.

Return portable Mermaid or the project's established format, followed by a legend, evidence paths or symbols, important omissions, uncertainties, and the next validation needed. Keep the diagram small enough to review.
