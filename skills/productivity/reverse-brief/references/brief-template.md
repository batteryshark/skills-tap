# Reverse brief template

1. Problem and target users
2. Observed user-visible behavior
3. Public interfaces and data contracts
4. Operational and platform constraints
5. Acceptance criteria with verification evidence
6. Inferred design decisions and confidence
7. Contradictions between docs, code, tests, and history
8. Unknowns and questions for the owner

Attach a file path or command result to each observed claim. Inferences should explain the evidence and at least one plausible alternative.
