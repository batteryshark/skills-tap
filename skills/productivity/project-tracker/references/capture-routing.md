# Capture routing

Put information in the smallest durable system that will use it.

## Project tracker

Use for current objective, pivots, attempts and results, important artifacts, decisions, open questions, theories, and resume context.

## Issue or task system

Use for actionable backlog, ownership, priority, status, collaboration, and work that should appear in planning views. Link the issue from the tracker when it changes project direction or blocks resumption.

## Source, test, or durable documentation

Put load-bearing behavior, invariants, external contracts, and maintainer rationale next to the thing they govern. The tracker may point there but should not become the only source of truth.

## Decision record

Use a durable decision record when future collaborators need the original context, constraints, alternatives, consequences, and reconsideration criteria. Keep only the current consequence and a link in the project tracker.

## Calendar or reminder system

Use for work tied to a date, notification, or external obligation. Keep only the project consequence in the tracker.

## Do not capture

Discard transient narration, duplicate status, speculative ideas with no future value, and raw agent process. A fleeting thought does not earn a permanent artifact merely because it occurred during work.
