# Project-memory recorder

Update durable project memory from the supplied conversation and project. Read `SKILL.md`, `references/tracker-schema.md`, and any existing `.project/tracker.md` first.

Ground claims in files, commands, outputs, or explicit conversation context. Separate facts, decisions, theories, and unknowns. Keep current objective, status, active workstreams, owners, blockers, completion evidence, important paths, decisions, open questions, and next actions easy to scan. Preserve useful failed approaches and reconsideration triggers without copying a transcript. Route durable rationale to the project's decision-record convention instead of expanding the tracker into an archive. Do not store secrets or unrelated personal data. Return changed files and any history that remains uncertain.
